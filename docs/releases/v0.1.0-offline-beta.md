# v0.1.0-offline-beta

`v0.1.0-offline-beta` marks the first complete offline/fake SAT download beta for CFDI Vault MX. It is a release milestone for local operation only; it does not enable live SAT access.

## What is complete

| Area | Status |
|---|---|
| Local setup checks | `cfdi-vault status` and `cfdi-vault doctor` support redacted local profile validation. |
| Offline download planning | `cfdi-vault download plan` validates profile-backed criteria without submitting a request. |
| Offline request submission | `cfdi-vault download request` returns deterministic synthetic request output. |
| Offline sync | `cfdi-vault download sync` runs the fake request, verification, package, metadata, and XML evidence pipeline locally. |
| Offline status readback | `cfdi-vault download status --job-id <job-id>` reads safe persisted aggregates from local storage. |
| CFDI status outcomes | Fake status/cancellation outcomes classify active, cancelled, not found, unauthorized, retryable, permanent, and unknown states. |
| Retry safety | Cancelled and not-available outcomes are terminal and do not create infinite retry loops. |
| Operations docs | Offline/local runbooks cover troubleshooting, backup/restore, observability, and the SAT real gate. |

## Offline command path

Use an editable install for local development:

```bash
python -m pip install -e ".[dev]"
```

Then run the offline/fake flow:

```bash
cfdi-vault status
cfdi-vault doctor
cfdi-vault download plan --profile <profile> --from 2024-01-01 --to 2024-01-31 --kind metadata --direction received
cfdi-vault download request --profile <profile> --from 2024-01-01 --to 2024-01-31 --kind metadata --direction received
cfdi-vault download sync --profile <profile> --from 2024-01-01 --to 2024-01-31 --kind cfdi --direction received
cfdi-vault download status --profile <profile> --job-id <job-id>
```

Use placeholders for profiles and dates in documentation. Do not paste local operator paths, RFCs, request identifiers, package identifiers, or XML content into issues or PRs.

## Safety boundary

This milestone intentionally does **not** include:

- live SAT authentication;
- e.firma use against SAT;
- real CFDI XML;
- real SAT metadata;
- real SAT ZIP packages;
- committed certificates, keys, passwords, tokens, or local profile files.

Issue #50 remains the human gate for any future controlled SAT metadata smoke test.
